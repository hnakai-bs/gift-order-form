# HTML スライドと PDF 出力を揃える（メモ）

`slide.html` でブラウザ表示と PDF の**サイズ・余白・レイアウト**をほぼ揃えるために入れた設定のまとめです（色の再現は対象外）。別ファイルを作るときもこの方針を再利用できます。

---

## 目的

- **1スライド = 1PDFページ**
- **画面のスライド（16:9）と PDF のページ比率・レイアウトを近づける**
- macOS / Safari / Chrome の印刷まわりの癖を踏まえた CSS にする

---

## 大きな方針

### 1. 用紙サイズは「A4 横」ではなく「16:9 ページ」

- 画面の `.slide` は **`aspect-ratio: 16 / 9`** の横長。
- **A4 横（297×210mm）**は 16:9 より**縦に長い**ため、そのまま PDF にすると上下に余白や縮小の差が出やすい。
- `@page` を **幅 297mm × 高さ `297 × 9 ÷ 16` mm（167.0625mm）** にし、**1ページ＝16:9** に近づける。

```css
@page {
  size: 297mm 167.0625mm;
  margin: 0;
}
```

- **紙の A4 に印刷**するときは、この PDF より縦が長いので、プリンタ側で縮小・フィットが必要になることがある（データ用 PDF では 16:9 ページの方が HTML に近い）。

### 2. 印刷時のスライド寸法を pt で固定（WebKit 対策）

- `body` を `width: 297mm` などで固定すると、macOS の印刷で**小さな箱が左上**に乗ることがある。
- 印刷時の `.slide` は **`width: 841.89pt`（≒297mm）**、**`height: calc(幅 × 9 / 16)`** のように **pt + calc** でページと一致させる。
- 画面用の **`width: min(100%, 1280px)`** や **`aspect-ratio`** は印刷で **`!important`** で打ち消す。

### 3. 画面と PDF で同じ flex の「型」

- **画面**: `justify-content: flex-start`、**`.slide-body { flex: 1 }`** で下の余白を body 側が預かる（上詰めのスライド）。
- **以前の PDF**: `justify-content: center` や本文幅を狭めると、**HTML と別レイアウト**になっていた。
- **現在**: 印刷も **上寄せ + `slide-body` を `flex: 1`** に揃える。**タイトルだけのスライド**は **`.slide--title-only` だけ中央寄せ**。

### 4. 余白・間隔は CSS 変数で画面と印刷を対応付ける

`.slide` 上で定義し、**画面は `clamp`、印刷は mm で上書き**する。

| 変数 | 画面（例） | 印刷（例） |
|------|------------|------------|
| `--slide-pad-y` | `clamp(28px, 3.2vw, 48px)` | `9mm` |
| `--slide-pad-x` | `clamp(36px, 4vw, 64px)` | `12mm` |
| `--slide-header-gap` | `clamp(20px, 2.5vw, 36px)` | `6mm` |
| `--slide-section-gap` | `clamp(20px, 2.2vw, 32px)` | `5mm` |
| `--slide-row-gap` | `clamp(16px, 2vw, 32px)` | `4mm` |

新しいスライドを増やすときも、**まずこの変数だけ触る**と画面と PDF のバランスを保ちやすい。

### 5. グリッド比・行間（サイズまわり）

- 2カラム: **`grid-template-columns: 1fr 2.35fr`**（印刷でも同じ比で、列幅のバランスを HTML に合わせる）。
- 本文: **`line-height: 1.75`** を画面と揃え、縦方向の詰まり具合を近づける。

---

## 印刷ダイアログの推奨（目安）

- **用紙**: 可能ならドキュメント／アプリに合わせる（16:9 用紙を解釈できる場合）。
- **向き**: **横（ランドスケープ）**。縦だと横長レイアウトが潰れる。
- **余白**: **なし**または最小。
- **スケール**: **100%**（「ページに合わせる」だけ縮小されてズレることがある）。

---

## PDF をコマンドで出す（Chrome）

再現性が高いです。

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless --disable-gpu \
  --print-to-pdf="/path/to/out.pdf" \
  --no-pdf-header-footer \
  "file:///path/to/slide.html"
```

---

## つまずきやすかった点（簡易チューニング履歴）

1. **A4 横のみ** → 16:9 スライドと**縦比率が違う**ので上下に余白や違和感。
2. **`body` を mm 固定** → 一部環境で**小さく左上**に収まる → **幅 100% + `.slide` を pt** に変更。
3. **印刷だけ縦中央・幅狭め** → HTML と**全く別見え** → **画面と同じ flex（上寄せ・body flex:1）** に統一。
4. **`.slide-body { flex: 1 }` が印刷でも効かず下だけ真っ白** → 印刷側で **`flex-grow: 1`** を明示して画面と同じに。
