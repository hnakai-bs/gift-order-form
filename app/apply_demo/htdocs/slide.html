ご進物申込・申し込みシステム

お中元・お歳暮などの申込をオンライン化するご提案
（画面デモ・見積もり前提条件に基づく整理）

gift-order-form / apply_demo

01

背景と目的

現在紙の申込用紙で受け付けているご進物のお申し込みを、Web 上で入力・確認・送信できる仕組みへ移行することを目的としています。

  • お客様負担の軽減（再入力の削減、履歴の活用）
  • 受付オペレーションの整理（データの一元化、CSV 等での活用）
  • まずはミニマムな機能でオンライン申込をスタートし、運用しながら段階的に拡張する方針です。
02

システムで実現すること

  • お客様がオンライン画面から申し込み内容を入力できる
  • 入力内容を確認して送信できる
  • 過去の申し込み履歴を参照し、同様の内容で再注文したり、送り主情報を整えたうえで注文できる

初期対応では、過去データを指定フォーマット（Excel / CSV）でご提供いただき、取り込み・照会の土台とする想定です。

03

利用者向けの画面構成

画面 内容
ログイン・会員登録 電話番号・パスワード、新規会員登録、登録情報の追加・更新
ご注文一覧 履歴一覧、新規ご注文・再注文・マイページへの導線
新規・再注文 お届け先・商品・お届け希望日時（第三希望まで）・お支払いの入力
確認・完了 内容の最終確認、送信完了メッセージ
04

運用者向け管理画面

初期スコープでは、次の3 機能ブロックに限定して想定しています。

機能 内容
ログイン 運用者認証
ユーザー管理 会員の一覧・参照、パスワード変更、CSV 出力
申込管理 申込の一覧・参照、CSV 出力

商品マスタの編集は本画面からは行わず、別途の運用（ファイル連携等）を想定します。

05

初期スコープと今後の拡張

初期に含める想定

  • オンライン申込フロー一式
  • 管理画面（ログイン・ユーザー・申込）
  • レスポンシブで最低限利用可能な表示

要件外（フェーズで協議）

  • 在庫のシステム管理・発送ステータス連携
  • 利用者によるパスワードのセルフ変更
  • メールアドレスの有効性検証、申込取り消し機能
  • 商品検索・画像ギャラリー、手の込んだモバイル最適化

ブラウザは Safari / Chrome 最新版、管理者アカウントは初期おおよそ 3 件（SE 対応で作成）を想定しています。

07

本リポジトリのデモについて

apply_demo/htdocs 配下の HTML は、上記の画面イメージ・導線を静的デモとしてまとめたものです。実装仕様・API・本番データ連携は含みません。

  • 利用者：ログイン、会員登録、注文一覧、新規／再注文、確認画面など
  • 管理者：admin/ のログイン、ユーザー管理、申込一覧・詳細

詳細な画面デザイン・項目定義・データ連携は要件定義で確定後、開発見積の再提出（再見積もり）を前提としています。
